Realtime Voice Inference

Performance

The system is optimized for responsiveness during 1:1 video chat by reducing prompt orchestration overhead, minimizing blocking operations, and starting F5-TTS synthesis as early as possible during streaming generation.

A heavier runtime configuration with chain of thought reasoning, richer system prompts, and a more sophisticated TTS model like VibeVoice-1.5B trades latency for conversational richness. As a matter of personal taste, low latency and fast response time consistently felt better than higher quality outputs.

Benchmarks

I replaced toy prompts with memory benchmarks built from the actual transcript corpus. The test set uses roughly 75,000 real speaker-labeled transcript segments from 563 All-In Podcast episodes, then injects controlled memory facts into realistic conversation history.

The goal was not to prove the model can answer trivia. It was to measure the practical cost of conversational memory: how much prior context can be injected before latency becomes unacceptable, and whether buried facts remain retrievable.

This established the latency curve. The second question was whether the model could actually recover information buried inside that context.

Retrieval TestObserved ResultFinding
Explicit user memory~100%USER: I like turtles. was recovered reliably. Clearly marked user facts were easy for the model to retrieve.
Targeted fact bank~100%With specific questions, the model recovered names, numbers, preferences, and identifiers such as Blue Mango, 207 podcast episodes, and the airspeed velocity of an unladen swallow.
Embedded sentence20–60%When the fact was hidden inside an ordinary transcript line, retrieval became much harder. The model had to separate the inserted memory from normal podcast noise.
Anomaly promptUnstableAsking "what was unusual?" measured salience, not memory. The transcript already contains jokes, fragments, and pivots, so the model often picked a different weird sentence.

The useful conclusion was that memory survived better than expected. The model could usually recover explicitly marked or clearly cued facts from realistic transcript context. The harder problem was not raw recall; it was attention and salience.

For the realtime product, this makes the design constraint clearer: the practical memory limit is user patience and latency, not whether Qwen can technically ingest the context. The current runtime therefore favors smaller sliding windows for voice interaction, with larger context reserved for slower text or analysis modes.
